# override
Методы которые мы переопределяем в производном классе должны помечаться как override, чтобы компилятор понимал, что этот метод мы переопределяем, а не создаем новый. 

```cpp
class A{
public:
	virtual void smth();
}

class B : public A
{
public:
	void smth();//так мы переопределяем метод, т.к. сигнатуры схожи
}
```

Но если в базовом классе она изменится, то мы уже будем создавать свой собственный метод
```cpp
class A{
public:
	virtual void smth(int a);
}

class B : public A
{
public:
	void smth();//определяем свой метод
}
```

Чтобы этого избежать помечаем переопределяемый метод как override, теперь если сигнатура в базовом классе меняется, компилятор не может найти переопределяемый метод в производном классе и выдает ошибку.
